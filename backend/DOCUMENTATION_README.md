7. **`Documentation/MAINTENANCE_SUPPORT.md`** - Maintenance, backup, error handling

## How to Use

1. **For New Developers/Onboarding:**
   - Start with `Documentation/KT_README.md`
   - Read `Documentation/ARCHITECTURE_DOCUMENTATION.md`
   - Follow `Documentation/SETUP_DEPLOYMENT_GUIDE.md`

2. **For Development:**
   - Use `Documentation/API_REFERENCE.md` for API endpoints
   - Use `Documentation/DATABASE_SCHEMA.md` for database queries

3. **For Deployment:**
   - Follow `Documentation/SETUP_DEPLOYMENT_GUIDE.md`

4. **For Maintenance:**
   - Use `Documentation/MAINTENANCE_SUPPORT.md`
   - Use `Documentation/TESTING_GUIDE.md` for tests

5. **For Understanding Features:**
   - See `Documentation/FEATURE_USER_GUIDE.md`

---

All documentation is centralized in the `Documentation/` folder for easy access and knowledge transfer.
